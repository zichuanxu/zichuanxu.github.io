10,000 个训练 step 之后，这个小型语言模型在 TinyStories 验证集上的 loss 降到 0.8714，perplexity 为 2.39。它已经能生成局部连贯的句子，但还不能稳定维持完整故事结构。这个结果足以验证从 tokenizer 到 sampling 的链路能够工作，也清楚暴露了模型规模、上下文长度和评估方式的限制。

## 从文本到生成的完整链路 {#pipeline}

Assignment 1 的价值在于把语言模型拆回最基本的部件。输入文本先经过字节级 BPE，token ID 进入 Decoder-only Transformer，训练目标是预测下一个 token，最后再通过采样策略把 logits 还原成文本。

<div class="article-flow article-flow-wide" role="list" aria-label="语言模型训练链路">
  <span role="listitem">Raw bytes</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">BPE tokenizer</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">Token IDs</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">Decoder-only Transformer</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">Sampling</span>
</div>

每个部分都可以单独通过测试，但真正容易出错的是边界：special token 是否参与 merge、mask 是否在 softmax 前应用、checkpoint 是否包含优化器状态，以及 encode 与 decode 是否保持可逆。

## 让 BPE 训练从正确变得可用 {#bpe}

最直接的 BPE 实现会在每轮 merge 后重新扫描全部 pair。它容易验证，但总成本接近 $O(MP)$：$M$ 是 merge 次数，$P$ 是当前候选 pair 数量。语料稍大后，寻找最高频 pair 和重建计数都会成为瓶颈。

实现经历了三步：

1. **全量计数基线。** 把文本预分词成 token tuple，统计相邻 pair 的全局频率，每轮选择频率最高的 pair。
2. **最大堆与延迟删除。** Python 的 heapq 使用负频率模拟最大堆。pair 频率变化时不修改堆中旧记录，而是压入新记录；弹出时与当前计数核对，过期记录直接丢弃。
3. **受影响范围增量更新。** 建立 pair 到包含它的 word 的反向索引。一次 merge 只重算受影响的 word，而不是重新扫描全部语料。

encode 阶段也采用相同思路。用数组模拟双向链表后，合并 token 只需要修改相邻指针，不必反复移动 Python list。候选 pair 仍由堆维护，单个位置是否有效则由版本与存活标记判断。

这类“堆中允许旧值存在，使用时再验证”的策略很实用：它用可控的额外空间换掉了昂贵的中间元素更新，也让实现保持简单。

## Decoder-only Transformer 的最小结构 {#model}

模型没有使用原始 Transformer 论文中的 encoder–decoder 双塔，而是只保留用于自回归语言建模的 decoder 路径：

<div class="article-flow" role="list" aria-label="Decoder-only Transformer block">
  <span role="listitem">Embedding</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">RMSNorm</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">Causal MHA + RoPE</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">SwiGLU FFN</span>
  <i aria-hidden="true">→</i>
  <span role="listitem">LM Head</span>
</div>

注意力仍然是标准的 scaled dot-product attention：

$$
\operatorname{Attention}(Q,K,V)
=
\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d_k}} + M\right)V
$$

其中 $M$ 是 causal mask。未来 token 对应的位置在 softmax 之前被设为负无穷，因此当前位置只能看到自身和历史 token。

实现中使用 Pre-Norm：每个 attention 或 FFN 子层先做 RMSNorm，再经过残差连接。RoPE 只作用于 query 与 key，通过旋转角度把相对位置信息带入内积；value 仍保存要聚合的内容。FFN 使用 SwiGLU，把门控分支与内容分支逐元素相乘后投影回模型维度。

这些部件单独并不复杂，真正需要持续检查的是张量形状。多头注意力要在 $(B,L,D)$ 与 $(B,H,L,D/H)$ 之间切换，transpose 后再 view 前必须保证内存连续。

## 训练稳定性来自几个小约束 {#training}

训练目标是 next-token cross entropy。输入序列去掉最后一个 token，标签序列去掉第一个 token，再把 batch 与序列维度展平后计算 loss。Perplexity 是平均交叉熵的指数：

$$
\operatorname{PPL} = \exp\left(\frac{1}{N}\sum_{i=1}^{N} L_i\right)
$$

优化器使用 AdamW，把 weight decay 与梯度更新解耦。学习率先线性 warmup，再按余弦下降到最小值，训练后段保持不变。

<figure>
  <img src="/images/blog/cs336-assignment1/cosine-annealing.png" alt="先线性 warmup、再余弦下降并保持最小值的学习率曲线" loading="lazy" />
  <figcaption>学习率由 warmup、cosine annealing 和最小值保持三个阶段组成。</figcaption>
</figure>

此外还有三个容易遗漏的工程细节：

- 在 optimizer step 前做 global gradient clipping，限制异常 batch 造成的梯度爆炸；
- checkpoint 同时保存模型参数、优化器状态和当前 step，使恢复后的学习率与 AdamW 动量保持连续；
- 训练与验证使用完全分离的数据迭代器，评估时关闭 dropout 和梯度记录。

## 采样方法与结果边界 {#sampling}

训练结束后，模型输出的是下一个 token 的 logits。Greedy sampling 总是选择最大值，结果稳定但容易重复；Top-K 把候选限制在概率最高的 K 个 token；Top-P 选择累计概率达到阈值的最小集合。Temperature 则在 softmax 前缩放 logits：

$$
p_i(T)=\frac{\exp(z_i/T)}{\sum_j \exp(z_j/T)}
$$

<figure>
  <img src="/images/blog/cs336-assignment1/temperature.png" alt="不同 temperature 下同一组 token 的概率分布变化" loading="lazy" />
  <figcaption>Temperature 越低，分布越集中；越高，低概率 token 获得更多采样机会。</figcaption>
</figure>

当前结果能生成局部连贯文本，但没有系统测量重复率、故事一致性或不同采样参数的影响。下一轮实验最值得补充的不是单纯增加训练步数，而是固定 checkpoint，对不同 temperature、Top-K 与 Top-P 组合做同一组 prompts 的对照评估。

同样需要明确的是：这篇记录没有附带可运行仓库、依赖锁文件和硬件信息，因此尚不能构成完整的复现实验。公开结果应被理解为一次实现记录，而不是基准成绩。
